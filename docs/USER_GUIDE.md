## 1. 产品概述

### 1.1 产品定义

**AI Live Room** 是一款基于现代Web技术栈构建的**实时互动直播平台桌面客户端**，提供沉浸式的观看体验和丰富的社交互动功能。

### 1.2 核心价值主张

| 价值维度 | 具体体现 |
|----------|----------|
| **🎯 实时性** | 弹幕延迟 < 100ms，消息即时送达 |
| **🎨 沉浸感** | 赛博朋克视觉风格，霓虹渐变主题 |
| **🔗 社交化** | 多人实时互动，弹幕/礼物/表情 |
| **💻 跨平台** | Windows/macOS/Linux 三端支持 |
| **⚡ 高性能** | WebSocket长连接，60fps流畅动画 |

### 1.3 目标用户画像

#### 👤 观众用户
- **年龄**: 18-35岁
- **特征**: 游戏玩家、二次元爱好者、科技爱好者
- **需求**: 观看直播、发送弹幕、打赏主播、社交互动

#### 🎙️ 主播用户
- **年龄**: 20-40岁
- **特征**: 内容创作者、游戏主播、知识分享者**
- **需求**: 管理直播间、查看数据、互动粉丝

## 4. 核心功能详解

### 4.1 🔐 用户认证模块

#### 4.1.1 功能描述

提供安全可靠的用户身份验证机制，支持新用户自动注册和老用户快速登录。

#### 4.1.2 功能特性

| 特性 | 描述 | 技术实现 |
|------|------|----------|
| **用户注册** | 首次登录时自动创建账户 | MySQL INSERT + UUID Token |
| **密码登录** | 可选密码字段（当前版本为可选） | SHA-256哈希存储 |
| **Token认证** | 登录后生成唯一会话令牌 | UUID v4 + Redis缓存 |
| **会话持久化** | 关闭浏览器后重新打开仍保持登录状态 | localStorage 存储 |
| **自动恢复** | 页面刷新后自动从本地恢复登录态 | Pinia Store 初始化 |

#### 4.1.3 使用方法

**操作步骤**:
1. 打开登录页面 `http://localhost:5174`
2. 定位到 **USERNAME** 输入框
3. 输入 desired username（3-50个字符）
4. （可选）在 **PASSWORD** 输入框输入密码
5. 点击紫色渐变按钮 **"Enter the Matrix"**
6. 等待加载动画完成（约1秒）
7. 成功后自动跳转至直播间

**输入规则**:
- 用户名：必填，3-50字符，支持中英文数字
- 密码：选填，如填写则需6-20字符
- 特殊字符：允许 `@`, `.`, `_`, `-`

**成功标志**:
- 按钮文字变为 "Connecting..." 并显示旋转图标
- URL地址变为 `/room/1`
- 页面显示直播间主界面

**失败处理**:
- 空用户名：显示红色提示 "请输入用户名"
- 网络错误：显示 "登录失败，请重试"
- API超时：10秒后自动提示超时

#### 4.1.4 数据流向图

```
[用户] --输入账号密码--> [LoginView.vue]
                              ↓
                    调用 userStore.login()
                              ↓
              [apiService] --POST--> [/api/auth/login]
                                        ↓
                              [Node.js API Server]
                                        ↓
                         ┌────────────────────────┐
                         │  1. 查询MySQL用户表      │
                         │  2. 不存在则INSERT新用户  │
                         │  3. 生成UUID Token      │
                         │  4. 更新用户Token字段    │
                         │  5. 返回用户完整信息      │
                         └────────────────────────┘
                                        ↓
                    返回 {success:true, data:{...}}
                              ↓
              [userStore] 更新状态 + 存储localStorage
                              ↓
           [router.push('/room/1')] 跳转到直播间
```

### 4.5 📊 实时数据统计

#### 4.5.1 在线人数统计

**数据来源**: WebSocket `online-count` 事件  
**更新频率**: 用户进出房间时实时更新  
**显示位置**: 
- 直播间右上角红色LIVE标签内
- 底部状态栏左侧

**计算逻辑**:
```javascript
// 后端实现
socket.on('join-room', async ({ roomId }) => {
  const onlineUsers = await io.in(`room-${roomId}`).fetchSockets()
  io.to(`room-${roomId}`).emit('online-count', {
    count: onlineUsers.length,
    roomId
  })
})
```

**数值含义**: 当前通过WebSocket连接到该房间的独立用户数（非累计PV）

#### 4.5.2 点赞数统计

**当前实现**: 基于初始随机值 + WebSocket增量更新  
**初始值**: 10000-60000之间随机  
**更新方式**: 收到其他用户弹幕时 +0~9  
**显示位置**: 视频区底部控制栏 ❤️ 图标旁

**未来改进**: 
- 对接真实点赞API
- 显示今日/总点赞趋势图
- 点赞排行榜

#### 4.5.3 弹幕总量统计

**计算方式**: 本地数组长度 `danmakuList.value.length`  
**显示位置**: 底部状态栏 💬 图标旁  
**上限**: 50条（超出自动清理旧消息）

## 8. FAQ 常见问题

### Q1: 忘记用户名怎么办？

**A**: 当前版本无需记忆用户名。每次打开应用时：
- 如果之前登录过，localStorage会保存上次用户名，自动填充
- 如果清除浏览器数据，直接输入任意新用户名即可（会自动注册为新账户）

### Q2: 可以在多台设备上同时登录吗？

**A**: ✅ 支持。但需要注意：
- 同一用户名在不同设备登录会生成不同的Token
- 目前没有"踢掉旧设备"机制
- 所有设备都会收到相同的弹幕广播

### Q3: 弹幕发送后别人能看到吗？

**A**: ✅ 能。当满足以下条件时：
- 你和其他用户在同一房间（目前只有房间ID=1）
- 对方也保持WebSocket连接在线
- 网络延迟正常（<200ms）

**验证方法**: 打开两个浏览器窗口，用不同用户名登录，在一个窗口发弹幕，另一个窗口应该能在1-2秒内看到。

### Q4: 为什么我的弹幕颜色每次都不同？

**A**: 这是设计如此。系统从5种赛博朋克配色中随机分配：
- `#00ff41` (荧光绿)
- `#ff006e` (霓虹粉)
- `#8338ec` (电紫)
- `#3a86ff` (赛博蓝)
- `#ffbe0b` (金黄)

目的是让弹幕流看起来更丰富多彩，避免视觉单调。

### Q5: 如何退出登录？

**A**: 当前版本UI暂无登出按钮，可以通过以下方式：
1. 清除浏览器localStorage（F12 → Application → Local Storage → Clear）
2. 或者直接关闭浏览器标签页

**计划**: v1.2版本会在右上角用户头像旁添加"Logout"按钮。

### Q6: 支持手机访问吗？

**A**: ⚠️ 勉强支持。技术上可以在手机浏览器打开，但体验不佳：
- 布局针对1920x1080桌面优化
- 手机屏幕太小会导致拥挤
- 触摸操作未做特殊优化

**推荐**: 使用电脑或平板获得最佳体验。手机专属版本规划在v2.0。

### Q7: 数据保存在哪里？安全吗？

**A**: 数据存储位置：

| 数据类型 | 存储位置 | 安全性 |
|----------|----------|--------|
| 用户信息（用户名/头像） | MySQL数据库 | ✅ 加密传输，密码哈希存储 |
| 会话Token | localStorage + Redis | ⚠️ 本地明文，但仅含非敏感信息 |
| 弹幕内容 | MySQL danmaku_messages表 | ✅ 持久化存储，支持审计 |
| 在线状态 | 内存（Socket.io连接） | ⚠️ 服务重启后丢失（符合设计） |

**隐私声明**: 当前版本不收集任何个人信息，用户名仅为展示用途。

### Q8: 如何部署到生产环境？

**A**: 请参考技术文档的"部署指南"章节（TECHNICAL_FRAMEWORK.md）。简要步骤：

```bash

# 2. 配置环境变量

# 3. 使用PM2启动API服务器
pm2 start src/server.js -i max --name "api-server"

# 4. 配置Nginx反向代理（可选）
# 将 localhost:3000 代理到 domain.com
```
