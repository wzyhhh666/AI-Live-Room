# C++ 直播弹幕+房间系统 — 弹幕模块设计

## 1. 模块目标

弹幕模块负责接收、校验、过滤、广播和存储弹幕，是项目最核心的业务链路之一。

---

## 2. 弹幕发送与接收流程

### 2.1 客户端发送流程

1. 用户在前端输入弹幕。
2. 客户端把文本封装成 `MSG_DANMAKU`。
3. 通过 TCP 长连接发送给服务端。
4. 服务端完成协议解析、鉴权和限流。
5. 弹幕进入敏感词过滤和广播流程。

### 2.2 服务端接收流程

1. Reactor 收到可读事件。
2. 读取完整包并反序列化 Protobuf Body。
3. 交给工作线程执行业务校验。
4. 校验通过后进入房间广播。
5. 最终写入热数据和冷数据存储。

---

## 3. 弹幕广播策略

### 3.1 房间内广播

- 默认只广播给当前房间内在线用户。
- 广播目标来自 `room:online:{roomId}` 或本地连接映射。
- 广播前先确认用户是否仍在房间中。

### 3.2 全量推送 vs 增量推送

#### 全量推送

适用于：
- 进入房间时初始化最近弹幕。
- 房间断线重连后的状态恢复。
- 调试或管理后台展示。

#### 增量推送

适用于：
- 实时直播弹幕主链路。
- 高频场景下的常规消息分发。
- 只向新增消息推送最新一条，而不重复刷全部历史。

### 3.3 推荐选择

- **实时链路默认采用增量推送。**
- **全量推送只用于初始化和补偿。**
- 这样可以降低带宽和 CPU 消耗，也更符合直播业务。

### 3.4 广播实现路径

弹幕模块不直接操作 socket，而是通过以下路径触发网络层发送：

1. `DanmakuService::processDanmaku()` 完成过滤和限流检查。
2. 构造 `MSG_DANMAKU` 的 Header 和 Body。
3. 通过 `RoomService` 或 `ConnectionManager` 获取房间在线用户列表。
4. 调用 `ConnectionManager::broadcast(roomId, message)` 触发发送。
5. `ConnectionManager` 遍历目标 fd，并交给网络层写事件或直接发送。

### 3.5 广播路径依赖关系

| 模块 | 职责 | 对外接口 |
|------|------|----------|
| DanmakuService | 构造消息、决定广播内容 | processDanmaku() |
| RoomService | 提供房间在线用户列表 | getOnlineUsers(roomId) |
| ConnectionManager | 遍历 fd 列表、触发网络发送 | broadcast(roomId, msg) |
| EpollServer | 实际 socket 写入 | sendTo(fd, data) |

弹幕模块只依赖 RoomService 和 ConnectionManager 的接口，不直接调用 EpollServer，符合单向依赖原则。

## 4. 弹幕频率限制

### 4.1 单用户限流

建议默认规则：

- 每个用户 1 秒最多 1 条弹幕。
- 允许短时间 burst，但必须有上限。
- 触发限流后返回错误码并提示稍后再试。

### 4.2 房间级限流

建议默认规则：

- 单房间每秒允许的弹幕量设置上限。
- 当房间热度异常升高时，优先保证系统可用性。
- 通过 Redis 计数器或本地原子计数实现。

### 4.3 限流顺序

1. 先做用户级限流。
2. 再做房间级限流。
3. 最后才进入广播和落库。

---

## 5. 弹幕存储策略

### 5.1 热数据 Redis

- 最近 N 条弹幕存 Redis。
- 房间最近弹幕列表用于重新进入房间时快速回放。
- 热数据需要设置 TTL，避免无限增长。

### 5.2 冷数据 MySQL

- 所有正式弹幕记录最终写入 MySQL。
- 用于审计、回放、数据分析和运营报表。
- 冷数据是最终持久化结果。

### 5.3 写入顺序

推荐顺序：

1. 过滤通过。
2. 投递广播。
3. 异步写 Redis 热缓存。
4. 异步写 MySQL 冷存储。

### 5.4 容错原则

- 广播成功但冷存储失败时，要记录补偿任务。
- 存储失败不能阻塞主广播链路太久。
- 弹幕链路要优先保证实时性。

---

## 6. 弹幕模块接口建议

- `processDanmaku()`：处理一条弹幕。
- `broadcastDanmaku()`：向房间内用户广播。
- `saveDanmaku()`：保存弹幕记录。
- `checkDanmakuRateLimit()`：检查限流。
- `filterDanmakuContent()`：执行敏感词过滤。

---

## 7. 关键约束

- 弹幕模块不能直接操作底层 socket。
- 弹幕模块不能绕过协议层自己拼包。
- 弹幕模块必须先过敏感词过滤，再做广播和存储。
