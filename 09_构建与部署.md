# C++ 直播弹幕+房间系统 — 构建与部署

## 1. 目标

本文件定义项目如何构建、依赖如何锁定、配置如何加载、如何部署到运行环境。

---

## 2. CMakeLists.txt 结构

### 2.1 推荐目录

```text
chatroom-server/
├── CMakeLists.txt
├── cmake/
├── include/
├── src/
├── proto/
├── config/
├── tests/
├── tools/
└── third_party/
```

### 2.2 CMake 分层

- 根 CMake 负责全局选项和版本。
- `src/` 下按模块生成库或目标文件。
- `tests/` 单独构建测试目标。
- `tools/` 单独构建压测或运维工具。

### 2.3 目标拆分建议

- `chatroom_common`
- `chatroom_protocol`
- `chatroom_net`
- `chatroom_service`
- `chatroom_data`
- `chatroom_server`
- `chatroom_tests`

## 3. 第三方库依赖管理

### 3.1 推荐方案

- **vcpkg**：优先推荐，适合 C++17 工程统一拉取依赖。
- **Conan**：可作为备选，适合更复杂的依赖锁定场景。

### 3.2 锁定原则

- 所有第三方库必须锁版本。
- 不允许开发者本地手动“顺手升级”。
- 构建脚本和文档中的版本要一致。

### 3.3 建议依赖

- spdlog
- fmt
- protobuf
- gtest
- yaml-cpp
- nlohmann/json
- hiredis
- mysqlclient 或 mysql connector

---

## 4. 编译选项

### 4.1 Release

- `-O2`
- `-DNDEBUG`
- `-Wall -Wextra -Wpedantic`

### 4.2 Debug

- 保留调试符号 `-g`
- 开启额外警告
- 便于配合 gdb / lldb / VS 调试

### 4.3 平台建议

- Linux 下优先用 GCC + Ninja。
- Windows 下优先 MSVC + Visual Studio Generator。

---

## 5. 运行环境要求

### 5.1 Linux 推荐环境

- Ubuntu 22.04 / 24.04
- GCC 13+
- CMake 3.26+
- Ninja 1.11+
- MySQL 8.0+
- Redis 7.x

### 5.2 Windows 推荐环境

- Visual Studio 2022
- MSVC 2022+
- CMake 3.26+
- vcpkg

## 6. 配置文件格式

### 6.1 推荐格式

- 运行配置优先使用 **JSON** 或 **YAML**。
- 两者不要混用同一类配置。
- JSON 更适合机器读取，YAML 更适合人工维护。

### 6.2 建议配置项

- 服务器监听地址和端口
- MySQL / Redis 连接信息
- 心跳间隔和超时
- 线程池大小
- 日志级别
- 敏感词库路径
- 房间缓存刷新策略

## 7. 部署步骤

1. 安装依赖。
2. 生成构建目录。
3. 执行 CMake 配置。
4. 编译服务器和测试程序。
5. 准备配置文件和数据库。
6. 启动服务并验证连接。

---

## 8. 部署约束

- 构建脚本必须可重复执行。
- 不能把构建依赖写死在某个人电脑上。
- 所有配置必须能通过文件或命令行覆盖。
